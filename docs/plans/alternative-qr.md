# Alternative QR Code Generation Plan

## Goal
Provide a reliable way for the user to generate a valid QR code for linking `signal-cli`, bypassing terminal rendering issues.

## Problem
The user's terminal is likely distorting the QR code generated by `qrencode` (aspect ratio or color issues), making it scan as "invalid" by the Signal app.

## Proposed Solution
Create a Python script `scripts/generate_qr.py` that:
1.  Takes the `sgnl://` URI as an argument.
2.  Uses the `qrcode` python library to generate a high-contrast ASCII art QR code OR saves it to a PNG file.
3.  Since we are in a headless/terminal environment, we will try a better ASCII generation method first.

## Changes
### `flake.nix`
- Add `python3` and `python3Packages.qrcode` to `buildInputs`.

### `scripts/link_device.sh`
- Update to use the python script for generation instead of `qrencode`.

## Verification
- User manual verification (scanning the code).
